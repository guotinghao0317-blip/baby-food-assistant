# 需求：给全部页面做手机端适配

## 需求背景
当前项目已经完成核心功能开发，但现有页面在手机端浏览体验不好，需要专门做移动端适配。

## 需求描述
给宝宝辅食项目的**全部页面**进行手机端适配，确保在手机（360px - 414px宽度）上有良好的浏览体验。

## 需要适配的页面列表

1. **首页** `/`
2. **登录页** `/login`
3. **注册页** `/register`
4. **宝宝信息收集页** `/onboarding`
5. **营养分析页** `/nutrition/[baby_id]`
6. **食谱生成页** `/recipes/generate`
7. **食谱详情页** `/recipes/[id]`

## 适配要求

### 布局要求
- 使用 Tailwind CSS 响应式类 (`sm:`, `md:`)
- 在手机端：
  - 左右padding缩小，充分利用屏幕宽度
  - 网格布局从多列改成单列
  - 大标题适当缩小
  - 按钮可点击区域足够大（至少 48x48px）

### 交互要求
- 表单输入框适合手机输入法弹出
- Tab栏在手机端可以横向滚动
- 没有横向滚动条
- 点击按钮有反馈效果

### 样式要求
- 保持和现有设计风格一致（粉色主题、渐变背景、圆角卡片）
- 文字大小适合手机阅读（最小不小于14px）
- 间距合理，不要太挤也不要太松

## 验收标准

- [ ] 所有列出的页面都已经适配
- [ ] 在375px宽度手机上完全正常显示
- [ ] 没有横向滚动条
- [ ] 表单可以正常输入，按钮可以正常点击
- [ ] Tab栏可以横向滑动
- [ ] 保持原有功能完全正常，不破坏桌面端体验

## 优先级
P0 - 必须全部完成，用户现在需要上线使用

## 影响范围
- 只改前端样式和布局
- 不改变后端逻辑
- 不改变业务功能
