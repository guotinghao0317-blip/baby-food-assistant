# 辅食助手项目 - 二期优化需求

## 项目背景
当前一期核心流程已经跑通：
- 用户注册登录 → 填写宝宝信息 → 营养分析 → 确认生成食谱 → 查看食谱
- 基础功能可用，但用户体验和信息展示需要优化

---

## 需求列表

### 需求一：营养分析页面详情优化

**原问题**：当前营养推荐展示过于笼统，用户看不懂为什么是这个量，也不知道该怎么注意饮食。

**改进要求**：

1. **每个营养素展示完整信息**
   - 营养素名称 + 当前推荐摄入量
   - **参考标准说明**：根据哪个月龄区间，参考标准值是多少
   - **文献依据**：明确说明参考 `WHO 婴幼儿营养指南` + `中国营养学会 0-2岁婴幼儿喂养指南`
   - **饮食建议/注意事项**：告诉家长应该侧重吃什么食物来补充，需要注意什么

2. **视觉改进**
   - 每个营养素用独立卡片展示
   - 增加参考标准区域（白色背景）
   - 增加建议区域（浅蓝色背景突出）
   - 关键营养素（如铁）需要高亮提醒

---

### 需求二：营养分析页面增加 AI 流式输出感受

**原问题**：当前页面是一次性全展示出来，没有 AI 生成的感觉。

**改进要求**：

1. 页面加载完成后，**营养分析内容逐步流式输出**
   - 先输出宝宝基本信息卡片
   - 然后逐个营养素卡片依次输出
   - 每个卡片有淡入动画效果
   - 给用户一种「AI正在为你分析」的感受

2. **输出顺序**：
   1. 宝宝基本信息卡片 → 已就绪，立刻显示
   2. 能量 → 等待 ~300ms
   3. 蛋白质 → 等待 ~300ms
   4. 脂肪 → 等待 ~300ms
   5. 碳水化合物 → 等待 ~300ms
   6. 铁 → 等待 ~300ms （高亮提醒）
   7. 钙 → 等待 ~300ms
   8. 维生素D → 等待 ~300ms
   9. 最后显示「分析完成」卡片

---

### 需求三：食谱展示页面改进 - 增加多样性和更好的交互

**原问题**：
1. 默认食谱内容重复性太高，显得单调
2. 当前全部平铺展示，一周7天都铺开太长了

**改进要求**：

1. **增加食谱多样性**：
   - 默认食谱需要更多样化，避免重复相同菜品
   - 不同月龄阶段应该有更匹配的菜品选择
   - 一周七天尽量不重复相同早餐/主菜

2. **生成过程流式输出**：
   - 生成食谱时，**一天一天依次输出**，而不是一次性全部出来
   - 每天作为一个卡片，输出完一天再输出下一天
   - 给用户「AI正在思考并生成每一天食谱」的感受

3. **交互框架改为 Tab 切换**：
   - 一周七天改用 **tab 标签页切换** 展示
   - 顶部显示周一到周日七个 tab
   - 用户点击哪个 tab 看哪天
   - 解决页面太长需要滚动的问题，体验更好

---

### 需求四：食谱菜品替换功能 - 「换一道菜」

**原问题**：用户生成食谱后，如果某一道菜宝宝不爱吃/不合适，没办法调整，只能就这样了。

**改进要求**：

1. **每道菜右下角增加「换一道」按钮**
   - 点击按钮触发重新生成这一道菜
   - 保持这道菜原来的餐次（早餐/午餐/晚餐/加餐）不变
   - 要求换一道**营养相似但食材不同**的菜品

2. **替换逻辑**：
   - 根据原来这道菜的营养定位（比如"红肉补铁"）
   - 推荐一道营养成分相似，但食材不同的替代菜
   - 保持推荐理由、烹饪步骤完整结构

3. **生成方式**：
   - 新菜品也是流式输出/逐步生成
   - 替换完成后页面更新这道菜的内容

4. **入口位置**：
   - 在菜品卡片的右上角或右下角放置「🔄 换一道」按钮

## 设计思路

### 营养分析卡片结构
每个营养素卡片包含：
```
┌─────────────────────────────────────────┐
│  营养素名称  [状态标签]                   │
│  推荐值 X 单位  |  参考标准 Y 单位          │
│  进度条                                    │
├─────────────────────────────────────────┤
│  📋 参考依据：                             │
│  根据WHO+中国营养学会，X月龄推荐...        │
├─────────────────────────────────────────┤
│  💡 饮食建议：                             │
│  具体建议文字...                          │
└─────────────────────────────────────────┘
```

### 流式输出交互
- 使用 useState 记录已渲染的营养素索引
- useEffect 中用 setTimeout 逐个递增
- 每个已渲染的卡片加 fade-in 动画

### 食谱 Tab 结构
```
┌─────────────────────────────────────────┐
│ [周一] [周二] [周三] [周四] [周五] [周六] [周日] │
├─────────────────────────────────────────┤
│                                         │
│         当前选中那天的详细食谱              │
│                                         │
└─────────────────────────────────────────┘
```

## 验收标准

- [ ] 每个营养素都有参考依据说明
- [ ] 每个营养素都有饮食建议
- [ ] 营养分析页面加载后逐个卡片依次出现，有动画效果
- [ ] 关键营养素（铁、维生素D）有高亮提醒
- [ ] 0-6个月营养素显示「此阶段主要从奶中获取」+ 建议
- [ ] 食谱生成过程一天一天逐步输出
- [ ] 食谱展示改用 Tab 切换，不用平铺
- [ ] 默认食谱多样性改进，减少重复菜品
