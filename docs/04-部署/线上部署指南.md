# 宝宝辅食助手 - 线上部署指南

本文档说明如何将项目部署到腾讯云服务器，支持 Nginx 反向代理和 Let's Encrypt SSL 证书。

## 系统要求

- 服务器：腾讯云 CentOS/Ubuntu
- Docker 和 Docker Compose 已安装
- 域名已备案并解析到服务器 IP
- 开放 80 和 443 端口

## 部署步骤

### 1. 克隆代码到服务器

```bash
git clone <your-repo-url>
cd 宝宝辅食项目
```

### 2. 配置环境变量

复制环境变量模板：

```bash
cp .env.example .env
```

编辑 `.env` 文件，修改以下配置：

```bash
# OpenAI API 配置
OPENAI_API_KEY=your-actual-api-key

# 安全配置
SECRET_KEY=$(openssl rand -hex 32)  # 生成随机密钥

# 运行环境
ENVIRONMENT=production

# 域名配置
DOMAIN=your-domain.com  # 替换为你的域名
NEXT_PUBLIC_API_URL=https://your-domain.com/api
```

### 3. 申请 SSL 证书

首次部署需要手动申请 SSL 证书，请执行：

```bash
# 启动 nginx 和 certbot
docker-compose up -d nginx certbot

# 申请证书（替换为你的域名和邮箱）
docker-compose run --rm certbot certonly --webroot \
  -w /var/www/certbot \
  -d your-domain.com \
  --email your-email@example.com \
  --agree-tos
```

证书申请成功后，重启 nginx：

```bash
docker-compose restart nginx
```

证书会自动续期，无需手动操作。

### 4. 启动所有服务

```bash
docker-compose up -d --build
```

### 5. 查看运行状态

```bash
docker-compose ps
docker-compose logs -f
```

## 目录结构说明

```

├── docker-compose.yml       # Docker Compose 配置

└── 线上部署指南.md          # 本文档
```

## 配置说明

### 端口映射

| 服务 | 端口 | 说明 |
|------|------|------|
| nginx | 80 | HTTP，自动跳转到 HTTPS |
| nginx | 443 | HTTPS |
| postgres | 5432 | PostgreSQL 数据库 |
| redis | 6379 | Redis 缓存 |
| backend | 8000 | 后端 API（仅内网访问） |
| frontend | 3000 | 前端（仅内网访问） |

### 反向代理路径

- `https://your-domain.com/` → 前端服务
- `https://your-domain.com/api/` → 后端 API

### SSL 证书

- 使用 Let's Encrypt 免费证书
- 证书有效期 90 天，certbot 容器自动续期
- 证书存储在 `./nginx/certbot/conf/` 目录，持久化保存

## 本地开发 vs 线上部署

**本地开发：**
```bash
# 直接使用原有的启动方式
docker-compose up -d
# 访问 http://localhost:3000
```

**线上部署：**
- 自动启用 HTTPS
- 通过 Nginx 反向代理访问
- 证书自动续期
- 后端和前端不直接暴露公网端口

## 故障排查

### 1. SSL 证书申请失败

检查：
- 域名是否正确解析到服务器
- 80 端口是否开放（防火墙/安全组）
- 目录权限是否正确

### 2. Nginx 启动失败

```bash
# 查看 nginx 日志
docker-compose logs nginx
```

检查配置文件中的域名是否正确替换。

### 3. 前端无法访问后端 API

检查：
- `NEXT_PUBLIC_API_URL` 环境变量是否正确设置
- 后端服务是否正常运行：`docker-compose ps backend`

### 4. 证书过期

检查 certbot 容器是否正常运行：
```bash
docker-compose ps certbot
# 手动续期
docker-compose run --rm certbot renew
```

## 安全建议

1. **数据库**：PostgreSQL 端口绑定到 127.0.0.1 更安全，如果需要外网访问请配置强密码
2. **密钥**：生产环境务必使用随机生成的 `SECRET_KEY`
3. **OpenAI Key**：保护好你的 API Key，不要泄露到代码仓库
4. **定期更新**：及时更新 Docker 镜像和依赖包

## 回滚到本地配置

如果需要回到本地开发模式，可以：

```bash
# 停止所有服务包括 nginx/certbot
docker-compose down
# 重新启动
docker-compose up -d postgres redis backend frontend
```

或者直接使用 git 恢复原来的 `docker-compose.yml`。
