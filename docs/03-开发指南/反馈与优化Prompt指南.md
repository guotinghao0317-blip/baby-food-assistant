# 反馈与优化 Prompt 指南

本项目为「辅食助手」：面向 2 岁以下宝宝的周食谱系统（信息收集 → 营养需求 → 食谱生成 → 对话调整）。技术栈：Next.js 前端、FastAPI 后端、PostgreSQL/Redis、Docker。

后续你在给出反馈时，只需在描述前加上对应**标签**，并复制下方对应类型的 **Prompt** 一起发给 AI，即可让 AI 按既定流程处理你的反馈。

---

## 标签一：`[Bug]` —— 问题定位

**何时使用**：发现现有流程出错、报错、表现异常，需要排查根因并给出修复思路时。

**复制以下整段作为 Prompt（可连同你的具体现象一起发）：**

```
【任务类型】Bug 问题定位

请按「问题定位」流程处理本次反馈。反馈时请同时提供「现象」与「预期效果」：

1. **复现路径**：从用户操作/接口调用出发，写出最小可复现步骤（例如：先点 X，再输入 Y，再点 Z）。
2. **现象**：当前实际表现（报错信息、界面表现、接口返回等）。
3. **预期效果**：在正常或正确情况下的期望表现或应有结果（便于对比问题根因）。
4. **可能原因**：结合代码/接口/配置，列出 2～3 个最可能的原因及对应位置（文件、函数、接口路径等）。
5. **修复建议**：针对最可能原因给出具体修改思路或步骤（含建议改动的文件与逻辑），便于我直接实施。

请先理解我下面描述的问题（含现象与预期效果），再按上述 1～5 点逐项输出；若信息不足，请指出需要我补充的环节（如：复现步骤、报错栈、截图、预期效果描述等）。
```

**示例用法**：
> `[Bug]` 在「生成一周食谱」那一步，点了按钮没反应，控制台也没有报错。**预期效果**：应跳转到食谱结果页或在当前页展示生成的食谱。

## 标签二：`[功能优化]` —— 现有功能补全与优化

**何时使用**：某块功能已有雏形，但体验不完整、逻辑有漏洞、或需要补充边界情况/新需求时。

**复制以下整段作为 Prompt（可连同你的具体需求一起发）：**

```
【任务类型】功能优化

请按「功能优化」流程处理本次反馈：

1. **现状与缺口**：根据我的描述，简要归纳当前该功能已做到什么、缺少或不足什么。
2. **场景与期望**：补全「在什么场景下、希望系统怎样表现」（包括正常流程与异常/边界情况）。
3. **需求清单**：把上述期望拆成可落地的需求点（每条一句话，尽量可验收），并标出建议优先级（高/中/低）。
4. **实现建议**：指出与这些需求最相关的模块/文件/接口，以及建议的改动方向（不要求写完整代码，只要方向清晰即可）。

若我描述不清，请主动问我：目标用户是谁、典型操作路径、必须满足的约束（如性能、兼容性）等，以便把「功能优化」写完整。
```

**示例用法**：
> `[功能优化]` 营养需求计算里，过敏原只支持单选，希望支持多选，并且要能区分「不能吃」和「少吃」。

## 标签三：`[新功能]` —— 全新功能需求

**何时使用**：要新增一块当前系统里还没有的能力（新页面、新流程、新对接等）时。

**复制以下整段作为 Prompt（可连同你的想法一起发）：**

```
【任务类型】新功能

请按「新功能」流程处理本次反馈：

1. **背景与目标**：用 1～2 句话说明「为什么要做这个功能、解决谁的什么问题」。
2. **目标用户与场景**：谁在什么情境下会用到（如：家长在周末备餐时、管理员在后台配置时）。
3. **核心流程**：用步骤列表写出主流程（从入口到结束），并标出涉及的前后端环节（页面/接口/数据表等）。
4. **验收标准**：列出 3～5 条「做到了即可视为该功能合格」的验收点，尽量可测试。
5. **与现有架构的衔接**：说明该功能会复用或依赖哪些现有模块（如登录、食谱生成、Redis 缓存等），以及是否有新表、新接口、新环境变量等。

若我只有模糊想法，请通过提问帮我收敛到上述结构；若已比较清晰，请直接按 1～5 输出一版需求说明，便于评审和开发排期。
```

**示例用法**：
> `[新功能]` 想加一个「食材库存」功能：家长录入家里现有食材，生成食谱时优先用库存里的，并提示缺哪些。

## 使用说明

| 你的诉求           | 使用标签       | 你要做的事 |
|--------------------|----------------|------------|
| 某个流程报错/异常  | `[Bug]`        | 在反馈前加上 `[Bug]`，并尽量提供：**现象**（实际表现）+ **预期效果**（期望结果），以及复现步骤、报错信息或截图；可一并复制上方「标签一」的整段 Prompt。 |
| 已有功能要增强、补全 | `[功能优化]`   | 在反馈前加上 `[功能优化]`，说明当前不足和期望效果；可一并复制「标签二」的整段 Prompt。 |
| 想加一个全新能力   | `[新功能]`     | 在反馈前加上 `[新功能]`，简述想法或目标；可一并复制「标签三」的整段 Prompt。 |

**快速调取方式**：
- 在 Cursor 里输入对应标签（如 `[Bug]`）+ 你的简短描述，再贴对应类型的 Prompt，即可让 AI 按该类型的流程回答。
- 若你只发标签 + 描述、不贴 Prompt，可说明「请按《反馈与优化Prompt指南》中 [Bug/功能优化/新功能] 的流程处理」，并附上本文件路径或本文件中的相关小节。
