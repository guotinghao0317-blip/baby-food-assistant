# 快速开始指南

## 📋 前置准备

### 1. 安装必要工具
- **Node.js 18+**：https://nodejs.org/
- **Python 3.10+**：https://www.python.org/
- **PostgreSQL 15+**：https://www.postgresql.org/download/
- **Redis**：https://redis.io/download/
- **Docker**（可选，推荐）：https://www.docker.com/

### 2. 获取 API Key
- **OpenAI API Key**：https://platform.openai.com/api-keys
  - 用于生成食谱和配图
  - 需要充值（建议先充值 $5-10 测试）

## 🚀 方式一：Docker 一键启动（推荐）

### 步骤

1. **克隆/进入项目目录**
```bash
cd /Users/jiayindeng/cursor尝试
```

2. **配置环境变量**
```bash
# 复制后端环境变量模板
cp backend/.env.example backend/.env

# 编辑 backend/.env，填入你的 OpenAI API Key
# OPENAI_API_KEY=sk-your-key-here
```

3. **启动所有服务**
```bash
docker-compose up -d
```

4. **查看日志**
```bash
docker-compose logs -f
```

5. **访问应用**
- 前端：http://localhost:3000
- 后端API文档：http://localhost:8000/docs

## 🛠️ 方式二：本地开发（手动启动）

### 后端启动

1. **进入后端目录**
```bash
cd backend
```

2. **创建虚拟环境**
```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
```

3. **安装依赖**
```bash
pip install -r requirements.txt
```

4. **配置环境变量**
```bash
cp .env.example .env
# 编辑 .env，填入：
# - DATABASE_URL（PostgreSQL连接字符串）
# - OPENAI_API_KEY
```

5. **初始化数据库**
```bash
# 确保 PostgreSQL 已启动并创建数据库
createdb babyfood_db  # 或使用 pgAdmin

# 运行应用（会自动创建表）
python main.py
```

6. **启动服务**
```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

后端将在 http://localhost:8000 运行

### 前端启动

1. **进入前端目录**
```bash
cd frontend
```

2. **安装依赖**
```bash
npm install
```

3. **配置环境变量**
```bash
cp .env.example .env.local
# 编辑 .env.local，确保：
# NEXT_PUBLIC_API_URL=http://localhost:8000
```

4. **启动开发服务器**
```bash
npm run dev
```

前端将在 http://localhost:3000 运行

### 1. 注册账号
- 访问前端页面
- 点击"注册"，输入邮箱和密码

- 点击"生成一周食谱"

## 🔧 常见问题

### Q1: 数据库连接失败
**解决**：
- 确保 PostgreSQL 已启动
- 检查 `DATABASE_URL` 配置是否正确
- 确认数据库 `babyfood_db` 已创建

### Q2: OpenAI API 调用失败
**解决**：
- 检查 `OPENAI_API_KEY` 是否正确
- 确认账户有余额
- 检查网络连接（可能需要代理）

### Q3: 前端无法连接后端
**解决**：
- 检查后端是否在 8000 端口运行
- 检查 `NEXT_PUBLIC_API_URL` 配置
- 查看浏览器控制台错误信息

### Q4: 配图生成失败
**解决**：
- 确认 OpenAI API Key 有 DALL-E 权限
- 检查账户余额（DALL-E 3 较贵）
- 可以暂时跳过配图，先测试文本功能

## 💡 开发建议

### 成本控制
- 开发阶段使用 `gpt-3.5-turbo` 而非 `gpt-4`（成本更低）
- 配图生成可以延迟加载（用户查看时再生成）
- 使用缓存避免重复生成

### 功能迭代
- **MVP 阶段**：先实现文本食谱生成，配图可后续添加
- **优化阶段**：添加知识库、RAG 检索
