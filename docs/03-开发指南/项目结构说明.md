## 🔑 核心文件说明

### 后端核心

#### `backend/main.py`
- FastAPI 应用入口
- 注册所有路由
- CORS 配置
- 数据库表初始化

#### `backend/app/models.py`
定义所有数据库表：
- **User**：用户表
- **Baby**：宝宝信息表
- **NutritionRequirement**：营养需求表
- **Recipe**：食谱表
- **RecipeDetail**：食谱详情表（每日每餐）
- **Conversation**：对话历史表

#### `backend/app/routers/`
API 端点：
- `auth.py`：`/api/auth/*` - 注册、登录、获取用户信息
- `babies.py`：`/api/babies/*` - 创建/获取/更新宝宝信息、获取营养需求
- `recipes.py`：`/api/recipes/*` - 生成食谱、获取食谱、调整食谱、历史列表
- `conversations.py`：`/api/conversations/*` - 发送消息、获取对话历史

#### `backend/app/services/`
业务逻辑：
- **nutrition.py**：根据宝宝信息计算营养需求（参考 WHO/中国营养学会标准）
- **recipe_generator.py**：调用 OpenAI API 生成一周食谱
- **recipe_adjuster.py**：根据用户反馈调整食谱
- **image_generator.py**：调用 DALL-E 生成配图

### 前端核心

#### `frontend/app/page.tsx`
- 首页展示
- 功能入口

#### `frontend/app/layout.tsx`
- 根布局组件
- 全局样式引入

## 🔄 数据流程

### 1. 用户注册/登录
```
前端 → POST /api/auth/register
     → POST /api/auth/login
     → 获取 Token
```

### 2. 添加宝宝信息
```
前端 → POST /api/babies
     → 后端创建 Baby 记录
     → 自动计算营养需求
     → 返回宝宝信息
```

### 3. 生成食谱
```
前端 → POST /api/recipes/generate
     → 后端获取宝宝信息和营养需求
     → 调用 recipe_generator.py
     → 调用 OpenAI API 生成食谱
     → 保存到数据库
     → 返回完整食谱
```

### 4. 调整食谱
```
前端 → PUT /api/recipes/{id}/adjust
     → 后端获取当前食谱
     → 调用 recipe_adjuster.py
     → 调用 OpenAI API 生成调整方案
     → 更新数据库
     → 返回调整后的食谱
```

## 🔐 安全注意事项

1. **环境变量**
   - 不要提交 `.env` 文件到 Git
   - 生产环境使用强随机 `SECRET_KEY`

2. **数据库**
   - 生产环境使用强密码
   - 定期备份

3. **API 限流**
