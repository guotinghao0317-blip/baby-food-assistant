# 宝宝辅食项目 - 主流程回归检查清单

本文档定义了每次新功能开发后，必须执行全量回归测试的核心主流程。

## 回归检查流程

按照以下顺序完整执行一遍所有步骤，确认功能正常：

---

### 1. 用户注册流程
- [ ] 打开 `/register` 注册页面
- [ ] 输入邮箱和密码（6-72个字符）
- [ ] 点击注册按钮
- [ ] 预期结果：注册成功，自动跳转到登录页

### 2. 用户登录流程
- [ ] 打开 `/login` 登录页面
- [ ] 输入注册的邮箱和密码
- [ ] 点击登录按钮
- [ ] 预期结果：登录成功，自动跳转到首页

### 3. 首页功能检查
- [ ] 首页各卡片正常显示
- [ ] "开始使用"按钮可点击
- [ ] 预期结果：点击后跳转到用户引导页 `/onboarding`

### 4. 添加宝宝信息流程
- [ ] 填写宝宝基本信息：姓名、月龄
- [ ] 可选填写：体重、身高
- [ ] 选择进食能力（辅食初期/中期/晚期）
- [ ] 选择出牙情况（未出牙/少量出牙/多数出牙）
- [ ] 选择过敏源（如有）
- [ ] 点击"完成"按钮提交
- [ ] 预期结果：保存成功，跳转到营养分析页 `/nutrition/{baby_id}`

### 5. 营养分析流程
- [ ] 宝宝基本信息卡片正确显示
- [ ] 各营养素卡片依次流式输出
- [ ] 参考标准和饮食建议正确显示
- [ ] 关键营养素（铁、维生素D）正确高亮标注
- [ ] 预期结果：分析完成后，"确认并生成一周食谱"按钮可点击

### 6. 生成食谱流程
- [ ] 点击"确认并生成一周食谱"
- [ ] 页面显示"正在生成食谱..."加载状态
- [ ] 等待生成完成
- [ ] 预期结果：生成成功，自动跳转到食谱详情页 `/recipes/{id}`

### 7. 查看食谱流程
- [ ] 顶部日期标签栏支持横向滚动（移动端）
- [ ] 可以点击切换不同日期
- [ ] 每日各餐菜品卡片正确显示
- [ ] 食材列表清晰可见
- [ ] 烹饪步骤编号正确
- [ ] 菜品图片正确显示（如果有）
- [ ] 营养估算信息正确显示
- [ ] 预期结果：切换流畅，内容显示完整不溢出

### 8. 换一道菜功能
- [ ] 在任意菜品卡片点击"🔄 换一道"按钮
- [ ] 显示加载状态
- [ ] 等待重新生成完成
- [ ] 预期结果：新菜品成功生成并替换原菜品，内容流式输出正常

---

## 移动端额外检查点

如果本次修改涉及前端样式/布局，在移动端还需要额外检查：

- [ ] 所有页面没有横向滚动条
- [ ] 所有按钮点击区域 ≥ 48px × 48px
- [ ] 旋转屏幕后布局正常
- [ ] 输入框不被软键盘遮挡
- [ ] 图片自适应容器，不溢出

## 通过标准

所有检查项全部通过，没有功能性问题，微小UI问题不影响使用可以通过。只要有一个核心功能不可用，本次回归不通过，必须修复后重新测试。
