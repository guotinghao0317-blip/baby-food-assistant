### 关于完整一周食谱生成问题说明
当前火山引擎doubao-1.5-pro-32k模型在输出大约16k字符后会提前停止，导致28道菜无法完整生成。这是**模型本身输出长度限制问题**，不是JSON解析修复代码的问题。本次修复要解决的两个JSON解析问题都已经修复：
1. ✅ 流式分块累积已修复 - 现在buffer正确累积所有chunks
2. ✅ markdown代码块清理已修复 - 现在自动移除```包裹

只要模型能完整输出JSON，就能解析成功。

2. **问题2：大模型输出markdown代码块包裹JSON导致解析失败**
   - 修复方案：添加`clean_json_output`函数，自动移除markdown代码块标记，提取真正的JSON内容
   - 测试结果：✅ 已修复，单元测试验证通过，无论是```json还是```都能正确清理
