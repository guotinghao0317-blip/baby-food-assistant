# 全网站移动端适配测试计划

## 1. 测试概述

### 1.1 测试范围
本次测试针对**宝宝辅食助手**全网站移动端响应式适配进行验证，覆盖所有前端页面：
- 首页 (`/`)
- 登录页 (`/login`)
- 注册页 (`/register`)
- 宝宝信息收集页 (`/onboarding`)
- 食谱生成页 (`/recipes/generate`)
- 食谱详情页 (`/recipes/[id]`)

### 1.2 改动说明
- 使用Tailwind CSS响应式工具类进行改造
- 采用移动优先 (Mobile First) 设计理念
- 修改了网格布局、按钮尺寸、字体大小等
- 适配断点：
  - 移动设备：< 768px
  - 平板设备：768px - 1024px
  - 桌面设备：> 1024px

### 1.3 测试环境
| 环境 | 浏览器/设备 | 屏幕尺寸 |
|------|-------------|----------|
| 开发环境 | Chrome 移动端模拟器 | 360px - 1920px |
| 线上验证 | iOS Safari | iPhone SE/12/13/14 |
| 线上验证 | Android Chrome | 多种分辨率 |
| 线上验证 | iPad 平板 | iPad Pro |

## 2. 测试重点

### 2.1 布局适配
- [ ] 容器宽度：在小屏幕上不出现水平滚动条
- [ ] 网格布局：移动端单列，平板多列自适应
- [ ] 间距：内边距/外边距在移动端适当缩小
- [ ] 溢出处理：内容超出屏幕是否正确处理

### 2.2 交互元素适配
- [ ] 按钮尺寸：移动端至少 44px × 44px 点击区域
- [ ] 表单输入：输入框尺寸适合触屏操作
- [ ] 可点击元素：间距足够，避免误触
- [ ] 单选框/复选框：点击区域足够大

### 2.3 文字排版
- [ ] 字体大小：移动端不小于 14px
- [ ] 行高：保证可读性
- [ ] 标题层级：正确缩放
- [ ] 文字不超出容器

### 2.4 特殊场景
- [ ] 横屏切换：适配正确
- [ ] 下拉刷新：浏览器原生正常工作
- [ ] 视口缩放：禁止用户缩放但保留双击手势
- [ ] 刘海屏/挖孔屏：内容不被遮挡

### 2.5 回归测试
- [ ] 桌面端原有功能不受影响
- [ ] 所有交互功能正常
- [ ] 表单提交正常

## 3. 测试策略

### 3.1 开发环境测试
- 视觉快照测试：对比不同断点下的布局渲染
- 响应式工具类单元测试：验证正确应用
- 窗口大小变化模拟：验证动态适配

### 3.2 线上环境测试
- 真机冒烟测试：各主流移动设备逐一验证
- 核心用户流程端到端测试：从登录到生成食谱完整流程

## 4. 通过标准
1. 所有页面在 360px ~ 414px 移动设备上正常显示，无水平滚动
2. 所有可点击元素尺寸 ≥ 44px × 44px
3. 文字大小 ≥ 14px，可读性良好
4. 核心功能流程在移动端可完整操作
5. 桌面端功能不受影响，布局保持原有效果
