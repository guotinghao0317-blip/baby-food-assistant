# 食谱详情页移动端适配说明

## 任务：MOB-006 食谱详情页适配

### 修改文件
- `/app/recipes/[id]/page.tsx` - 主页面文件

### 完成的适配内容

| 需求 | 实现方式 |
|------|----------|
| **餐次卡片单列显示** | 当前布局每餐已经是单列，本餐营养信息在移动端改为单列 `grid-cols-1 sm:grid-cols-2` |
| **图片自适应宽度** | 将固定高度 `h-48` 改为 `h-auto max-h-64`，保持 `w-full`，图片不溢出屏幕 |
| **文字大小≥16px** | 正文内容从 `text-sm` 改为 `text-base` (16px) |
| **按钮点击区域≥48px** | "换一道"按钮移动端改为 `h-12` (48px)，占满宽度；返回按钮增加内边距保证高度 |
| **保持左右内边距** | 页面根容器保持 `px-4` |

### 具体修改内容

1. **图片自适应**
   - `className="w-full h-auto max-h-64 object-cover rounded-md"`
   - 允许图片根据宽高比自适应，限制最大高度避免过大

2. **文字大小调整**
   - 推荐理由正文：`text-sm` → `text-base`
   - 食材标题：`text-sm` → `text-base`
   - 食材标签：`text-sm` → `text-base`
   - 烹饪步骤标题：`text-sm` → `text-base`
   - 烹饪步骤文字：`text-sm` → `text-base`
   - 烹饪步骤编号圆圈：增大尺寸从 `w-5 h-5` → `w-6 h-6`

3. **按钮点击区域优化**
   - "换一道"按钮：移动端 `h-12` (48px)，`w-full` 宽度，更容易点击
   - 返回按钮：增加padding到 `py-4` 确保最小高度48px
   - 左上角返回按钮：增加padding从 `p-2` → `p-3`，图标放大 `h-5 w-5` → `h-6 w-6`

4. **布局响应式**
   - 餐次头部标题从无 → `text-base sm:text-lg`
   - 餐次头部布局：`flex flex-col sm:flex-row`，移动端竖向排列按钮
   - 本餐营养信息：`grid-cols-1 sm:grid-cols-2`，移动端单列
   - 文字大小：日期信息 `text-base sm:text-sm`，移动端字号更大

5. **间距优化**
   - 烹饪步骤间距从 `space-y-1` → `space-y-2`，增加行间距提高可读性

### 验证
- TypeScript类型检查通过
- 所有功能保持正常工作
- 移动端适配完成，符合设计要求
