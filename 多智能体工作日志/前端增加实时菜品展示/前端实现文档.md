# 前端实时菜品展示实现文档

## 概述

在 `frontend/app/recipes/generate/page.tsx` 中实现了实时菜品展示功能，解决了以下问题：
1. 菜品生成后立即显示（打字机效果）
2. 生成完成后自动跳转到详情页
3. 通过天数Tab切换查看不同天的菜品

## 核心改动

### 1. 新增常量定义

```typescript
// 常量定义
const DAYS = ['周一', '周二', '周三', '周四', '周五', '周六', '周日']
const MEAL_TYPES: Record<string, string> = {
  breakfast: '早餐',
  lunch: '午餐',
  dinner: '晚餐',
  snack: '加餐'
}
```

**作用：**
- 提供统一的中文天数显示
- 提供餐次中文映射

### 2. 新增状态变量

```typescript
// 实时菜品展示相关状态
const [daysData, setDaysData] = useState<Record<number, any[]>>({}) // 存储每天生成的菜品数据
const [selectedDay, setSelectedDay] = useState(1) // 当前选中的天数 Tab
const [renderedDishes, setRenderedDishes] = useState(0) // 用于逐步入场动画
```

**作用：**
- `daysData`: 按天数存储生成的菜品数据，key为天数(1-7)，value为菜品数组
- `selectedDay`: 当前用户查看的天数，支持点击切换
- `renderedDishes`: 控制菜品逐步显示的数量，实现打字机效果

### 3. day_done 事件处理增强

**改动前：**
```typescript
} else if (data.type === 'day_done') {
  // 某一天生成完成
  console.log('[generateRecipe] day_done事件 - day:', data.day)
}
```

**改动后：**
```typescript
} else if (data.type === 'day_done') {
  // 某一天生成完成
  console.log('[generateRecipe] day_done事件 - day:', data.day, '菜品数:', data.details?.length)
  // 保存当天生成的菜品数据
  if (data.details && Array.isArray(data.details)) {
    setDaysData(prev => ({
      ...prev,
      [data.day]: data.details
    }))
    // 如果这是第一天或者当前选中的天，自动切换到这一天
    if (data.day === 1) {
      setSelectedDay(1)
    }
    // 启动打字机效果，逐步显示菜品
    setRenderedDishes(0)
    const totalDishes = data.details.length
    let count = 0
    const interval = setInterval(() => {
      count++
      setRenderedDishes(count)
      if (count >= totalDishes) {
        clearInterval(interval)
      }
    }, 300)
  }
}
```

**实现逻辑：**
1. 接收到 `day_done` 事件后，立即保存当天生成的菜品数据到 `daysData` 状态
2. 如果是第一天，自动切换到该天
3. 启动打字机效果：使用 `setInterval` 每300ms增加一个渲染的菜品数

### 4. finished 事件优化

**改动前：**
```typescript
setTimeout(() => {
  console.log('[generateRecipe] 跳转到食谱详情页')
  router.push(`/recipes/${data.recipe_id}`)
}, 800)
```

**改动后：**
```typescript
setTimeout(() => {
  console.log('[generateRecipe] 跳转到转谱详情页')
  router.push(`/recipes/${data.recipe_id}`)
}, 1500)
```

**优化点：**
- 延迟时间从 800ms 增加到 1500ms
- 确保所有 `day_done` 事件都已处理完毕后再跳转
- 给用户足够的时间看到生成的菜品预览

### 5. 实时菜品展示UI组件

新增了一个完整的实时菜品展示区域，包含：

#### 5.1 7天 Tab 切换器

```typescript
<div className="flex justify-center gap-2 mb-4 overflow-x-auto pb-2">
  {DAYS.map((dayName, index) => {
    const dayNum = index + 1
    const hasData = daysData[dayNum] && daysData[dayNum].length > 0
    const isSelected = selectedDay === dayNum

    return (
      <button
        key={dayNum}
        onClick={() => {
          setSelectedDay(dayNum)
          // 切换 Tab 时重置渲染动画
          if (hasData) {
            setRenderedDishes(0)
            // 逐步显示菜品
            const totalDishes = daysData[dayNum].length
            let count = 0
            const interval = setInterval(() => {
              count++
              setRenderedDishes(count)
              if (count >= totalDishes) {
                clearInterval(interval)
              }
            }, 300)
          }
        }}
        disabled={!hasData}
        className={`px-4 py-2 min-h-[48px] rounded-lg text-sm font-medium transition-all ${
          isSelected
            ? 'bg-pink-500 text-white shadow-md'
            : hasData
            ? 'bg-white text-gray-700 border border-gray-300 hover:bg-gray-50'
            : 'bg-gray-100 text-gray-400 cursor-not-allowed'
        }`}
      >
        {dayName}
        {hasData && <span className="ml-1 text-xs">✓</span>}
      </button>
    )
  })}
</div>
```

**特性：**
- 已生成的天数：白色背景，可点击，显示✓标记
- 选中的天数：粉色背景，高亮显示
- 未生成的天数：灰色背景，不可点击
- 点击切换时重新启动打字机动画

#### 5.2 菜品列表展示

```typescript
{daysData[selectedDay] && daysData[selectedDay].length > 0 ? (
  <div className="space-y-3">
    {daysData[selectedDay].map((dish: any, index: number) => {
      // 控制逐步显示
      if (index >= renderedDishes) {
        return null
      }

      return (
        <div
          key={dish.id || index}
          className="bg-gray-50 rounded-lg p-4 border border-gray-200 animate-fadeIn"
          style={{ animationDelay: `${index * 0.1}s` }}
        >
          <div className="flex justify-between items-start mb-2">
            <h4 className="font-medium text-gray-700">
              {MEAL_TYPES[dish.meal_type] || dish.meal_type}：{dish.dish_name}
            </h4>
          </div>

          {dish.ingredients && dish.ingredients.length > 0 && (
            <div className="mb-2">
              <p className="text-xs text-gray-500 mb-1">食材：</p>
              <p className="text-sm text-gray-600">
                {dish.ingredients.map((ing: any) => `${ing.name} ${ing.amount}`).join('、')}
              </p>
            </div>
          )}

          {dish.cooking_steps && dish.cooking_steps.length > 0 && (
            <div>
              <p className="text-xs text-gray-500 mb-1">烹饪步骤：</p>
              <p className="text-sm text-gray-600">
                {dish.cooking_steps.slice(0, 2).map((step: any) => step.description).join('；')}
                {dish.cooking_steps.length > 2 && '...'}
              </p>
            </div>
          )}
        </div>
      )
    })}
  </div>
) : (
  <p className="text-center text-gray-500">等待生成菜品...</p>
)}
```

**特性：**
- 只渲染 `index < renderedDishes` 的菜品，实现逐步显示
- 使用 `animate-fadeIn` CSS动画，配合 `animationDelay` 实现逐个淡入
- 显示菜名、食材、烹饪步骤（前2步）
- 简洁预览，不显示完整信息（完整信息在详情页）

## 交互流程

### 初始状态
1. 用户访问生成页面
2. 显示生成进度列表
3. 实时菜品展示区域隐藏（`Object.keys(daysData).length > 0` 为 false）

### 天菜生成中
1. 后端发送 `day_started` 事件
2. 前端更新当前步骤进度

### 某天完成
1. 后端发送 `day_done` 事件，包含菜品数据
2. 前端立即保存数据到 `daysData[day]`
3. 前端自动切换到第一天
4. 启动打字机动画：每300ms显示一道菜
5. 实时菜品展示区域显示出来

### 用户切换Tab
1. 用户点击某天Tab（必须有数据）
2. 更新 `selectedDay` 状态
3. 重置 `renderedDishes` 为0
4. 重新启动打字机动画，显示该天的菜品

### 全部部完成
1. 后端发送 `finished` 事件
2. 前端显示完成状态
3. 延迟1500ms后自动跳转到详情页
4. 用户在详情页可以看到完整的7天食谱

## 技术要点

### 1. 状态管理
- 使用 React Hooks 管理状态
- `useState` 管理本地状态
- `useRef` 管理不会触发重渲染的引用

### 2. 动画实现
- 使用 `setInterval` 实现定时器
- 每隔300ms更新 `renderedDishes` 状态
- React 重新渲染，显示下一个菜品
- 配合 CSS `animate-fadeIn` 实现淡入效果

### 3. 样式设计
- 使用 Tailwind CSS
- 保持与现有页面风格一致
- 响应式设计：`px-3 sm:px-4`、`text-xl md:text-2xl`
- 移动端适配：`min-h-[48px]` 确保按钮可点击区域足够大

### 4. 类型安全
- 使用 TypeScript
- `Record<number, any[]>` 定义数据结构
- 保留 `any` 类型以适应后端数据格式变化

### 5. 用户体验优化
- 实时反馈：菜品生成后立即显示
- 渐进式展示：打字机效果，不会突兀
- 可交互：支持切换查看不同天
- 视觉反馈：选中状态、完成状态、禁用状态
