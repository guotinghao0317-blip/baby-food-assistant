# 修复只生成2天的问题 - 修复摘要

## 问题描述

- **预期**：生成7天的完整食谱
- **实际**：只生成到周二（第2天）
- **现象**：状态从 `generated_days: 1` 突然变成 `status: 'completed'`

## 根本原因

1. **状态判断逻辑缺陷**：后端直接返回数据库状态，不验证数据量
2. **生成完成判断不准确**：`distinct()` 查询方式可能导致计数错误
3. **前端状态判断过于宽松**：只检查 `status` 不检查实际数据量

## 修复方案

### 后端修改

**文件1：`backend/app/routers/recipes.py`**

修改 `get_recipe_status` 端点：
- 使用 `.distinct().all()` 获取所有不重复的 `day_of_week` 值
- 转换为集合去重，确保计数准确
- **自动修复状态不一致**：
  - 如果 `status=completed` 但数据 < 7天，重置为 `generating`
  - 如果数据 >= 7天但 `status≠completed`，更新为 `completed`
- 添加详细日志

**文件2：`backend/app/services/recipe_generator.py`**

修改 `generate_next_day_sync` 函数：
- 使用相同的去重查询方式
- 添加详细日志记录每步生成后的状态
- 明确标记未完成的天数

### 前端修改

**文件：`frontend/app/recipes/[id]/page.tsx`**

修改 `startGeneration` 函数：
- **双重检查**：既要 `status === 'completed'`，也要 `generated_days >= 7`
- **检测状态不一致**：发现不一致时继续生成而不是中断
- 添加警告日志
