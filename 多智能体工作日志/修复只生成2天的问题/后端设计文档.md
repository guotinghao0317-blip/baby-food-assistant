# 修复只生成2天的问题 - 后端设计文档

## 问题描述

用户在食谱详情页遇到以下问题：
- 预期生成7天的完整食谱
- 实际只生成到周二（第2天）
- 控制台日志显示状态异常从 `generated_days: 1` 突然变成 `status: 'completed'`

### 日志分析

```
当前生成状态: {recipe_id: 8, status: 'generating', generated_days: 0, total_days: 7}
当前生成状态: {recipe_id: 8, status: 'generating', generated_days: 1, total_days: 7}
当前生成状态: {recipe_id: 8, status: 'completed', generated_days: 1, total_days: 7}
✅ 全部7天生成完成
```

**关键发现：**
1. 状态突然从 `generating` 变成 `completed`
2. `generated_days` 只增加到 1 就结束了
3. 前端错误地认为"全部7天生成完成"

### 数据分析

```
已生成的天数统计: {1: 7, 2: 4}
```

这说明：
- 周一（day=1）有 **7道菜**（异常，应该是4道）
- 周二（day=2）有 **4道菜**（正常）

## 根本原因

### 1. 状态判断逻辑缺陷

**原代码（recipes.py:388-416）：**
```python
@router.get("/recipe-status/{recipe_id}")
async def get_recipe_status(...):
    # 统计已生成的天数
    generated_days = db.query(RecipeDetail.day_of_week).filter(
        RecipeDetail.recipe_id == recipe_id
    ).distinct().count()

    return {
        "recipe_id": recipe.id,
        "status": recipe.status,  # 直接返回数据库状态
        "generated_days": generated_days,
        "total_days": 7
    }
```

**问题：**
- 直接返回数据库的 `status` 字段
- 如果数据库状态错误（如被错误地设置为 `completed`），前端会误认为已完成
- 没有校验 `status` 和实际数据量是否一致

### 2. 生成完成判断不准确

**原代码（recipe_generator.py:376-388）：**
```python
# 检查是否所有天都生成完成
total_generated = db.query(RecipeDetail).filter(
    RecipeDetail.recipe_id == recipe_id
).distinct(RecipeDetail.day_of_week).count()

if total_generated >= 7:
    recipe.status = "completed"
    db.commit()
    all_completed = True
```

**问题：**
- `distinct(RecipeDetail.day_of_week).count()` 在某些情况下可能返回不准确的结果
- 没有详细的日志追踪，难以排查问题
- 没有验证数据完整性（如某天重复生成导致计数错误）

### 3. 前端状态判断过于宽松

**原代码（page.tsx:317-326）：**
```typescript
if (statusData.status === 'completed' || statusData.generated_days >= 7) {
  console.log('✅ 全部7天生成完成')
  setOverallStatus('finished')
  await fetchRecipe()
  saveCache()
  break
}
```

**问题：**
- 只要 `status === 'completed'` 就停止，不检查实际数据量
- 导致前端在只有1-2天数据时就认为完成

## 修复方案

### 1. 修复后端状态查询端点

**新代码（recipes.py:388-438）：**

```python
@router.get("/recipe-status/{recipe_id}")
async def get_recipe_status(...):
    """
    查询食谱生成状态
    返回已生成天数和整体状态
    """
    recipe = db.query(Recipe).filter(
        Recipe.id == recipe_id,
        Recipe.user_id == current_user.id
    ).first()

    if not recipe:
        raise HTTPException(status_code=404, detail="Recipe not found")

    # 统计已生成的天数 - 使用更准确的查询方式
    # 先获取所有不同的 day_of_week 值
    day_nums = db.query(RecipeDetail.day_of_week).filter(
        RecipeDetail.recipe_id == recipe_id
    ).distinct().all()

    # 转换为集合去重
    generated_day_set = set(d[0] for d in day_nums)
    generated_days = len(generated_day_set)

    # 详细日志
    logger.info(f"[Status Query] Recipe {recipe_id}: status={recipe.status}, generated_days={generated_days}, day_set={sorted(generated_day_set)}")

    # 如果数据库状态是 completed，但实际数据不足7天，重置为 generating
    if recipe.status == "completed" and generated_days < 7:
        logger.warning(f"[Status Query] Recipe {recipe_id}: 状态不一致！status=completed 但只有 {generated_days} 天数据，重置为 generating")
        recipe.status = "generating"
        db.commit()

    # 如果实际数据已满7天，但状态不是 completed，更新为 completed
    if generated_days >= 7 and recipe.status != "completed":
        logger.info(f"[Status Query] Recipe {recipe_id}: 实际数据已满7天，更新状态为 completed")
        recipe.status = "completed"
        db.commit()

    return {
        "recipe_id": recipe.id,
        "status": recipe.status,
        "generated_days": generated_days,
        "total_days": 7
    }
```

**改进点：**
1. 使用 `.distinct().all()` 获取所有不重复的 `day_of_week` 值
2. 转换为集合去重，确保计数准确
3. 添加详细日志记录状态和数据集合
4. **自动修复状态不一致**：
   - 如果 `status=completed` 但实际数据 < 7天，重置为 `generating`
   - 如果实际数据 >= 7天但 `status≠completed`，更新为 `completed`

### 2. 修复生成完成判断逻辑

**新代码（recipe_generator.py:371-395）：**

```python
# 检查是否所有天都生成完成
# 使用更准确的查询方式：先获取所有不同的 day_of_week 值
day_nums = db.query(RecipeDetail.day_of_week).filter(
    RecipeDetail.recipe_id == recipe_id
).distinct().all()

# 转换为集合去重
generated_day_set = set(d[0] for d in day_nums)
total_generated = len(generated_day_set)

logger.info(f"[Algorithm Day {day}] 保存完成后的状态: 生成了{len(saved_details)}道菜，总共有{total_generated}天，天数集合={sorted(generated_day_set)}")

from app.models import Recipe
recipe = db.query(Recipe).filter(Recipe.id == recipe_id).first()
all_completed = False

# 只有当真的有7天不同的数据时，才设置为 completed
if total_generated >= 7:
    recipe.status = "completed"
    db.commit()
    all_completed = True
    logger.info(f"[Algorithm Day {day}] ✅ All 7 days completed! Day set: {sorted(generated_day_set)}")
else:
    logger.info(f"[Algorithm Day {day}] ⏳ Not completed yet. {total_generated}/7 days done. Next day needed.")

return {
    "day": day,
    "details": saved_details,
    "all_completed": all_completed,
    "generation_source": generation_source
}
```

**改进点：**
1. 使用相同的去重查询方式，确保计数一致性
2. 添加详细日志记录每步生成后的状态
3. 明确标记未完成的天数，方便追踪

### 3. 修复前端状态判断

**新代码（page.tsx:317-338）：**

```typescript
// 循环检查状态并生成下一天，直到全部完成
while (isMountedRef.current && !abortControllerRef.current?.signal.aborted) {
  // 查询当前状态
  const statusResponse = await fetch(
    `${API_URL}/api/recipes/recipe-status/${numericRecipeId}`,
    { headers: { 'Authorization': `Bearer ${token}` } }
  )
  const statusData = await statusResponse.json()
  console.log('当前生成状态:', statusData)

  // 双重检查：既要看 status，也要看实际的 generated_days
  const isReallyCompleted = statusData.status === 'completed' && statusData.generated_days >= 7
  if (isReallyCompleted) {
    // 全部完成
    console.log('✅ 全部7天生成完成')
    setOverallStatus('finished')
    // 刷新完整食谱数据
    await fetchRecipe()
    // 保存缓存
    saveCache()
    break
  }

  // 如果后端状态是 completed 但实际数据不足，说明有数据不一致问题
  if (statusData.status === 'completed' && statusData.generated_days < 7) {
    console.warn('⚠️ 状态不一致：status=completed 但数据不足，继续生成')
    // 继续生成，不中断
  }

  console.log(`当前已生成 ${statusData.generated_days} 天，继续生成下一天`)
  // 生成下一天
  await generateNextDay(numericRecipeId, token)
}
```

**改进点：**
1. **双重检查**：既要 `status === 'completed'`，也要 `generated_days >= 7`
2. **检测状态不一致**：如果发现不一致，继续生成而不是中断
3. 添加警告日志，方便排查问题

## 技术要点

1. **SQLAlchemy distinct() 的正确用法**：
   - `.distinct()` 不带参数会查询所有列的去重
   - `.distinct(RecipeDetail.day_of_week)` 指定字段去重
   - 使用 `.all()` 获取所有值后再用 Python 集合去重最可靠

2. **状态一致性保证**：
   - 后端自动检测和修复状态不一致
   - 前端双重检查，防止误判

3. **调试日志**：
   - 详细记录每步操作的状态
   - 记录数据集合，便于排查问题
