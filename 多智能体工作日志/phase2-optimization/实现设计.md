# 二期优化实现设计文档

## 整体流程变更

原有流程：
```
用户填写宝宝信息 → 直接生成食谱 → 展示食谱
```

新流程：
```
用户填写宝宝信息 → 营养分析页面（流式展示）→ 用户确认 → 生成食谱 → 食谱详情（Tab切换+流式）
```

---

## 需求一 + 需求二：营养分析页面

### 新增文件
- `frontend/app/nutrition/[baby_id]/page.tsx` - 营养分析页面组件

### 设计要点

1. **营养素卡片结构**：
   ```
   ┌─────────────────────────────────────────┐
   │  营养素名称  [状态标签]                   │
   │  推荐值 X 单位  |  参考标准 Y 单位          │
   ├─────────────────────────────────────────┤
   │  📋 参考依据：                             │
   │  根据WHO+中国营养学会，X月龄推荐...        │
   ├─────────────────────────────────────────┤
   │  💡 饮食建议：                             │
   │  具体建议文字...                          │
   └─────────────────────────────────────────┘
   ```

2. **配置化营养素信息**：
   - 在代码中维护 `NUTRIENT_CONFIG`，每个营养素包含：名称、单位、各月龄参考标准、饮食建议、是否高亮
   - 这样方便后续维护修改

3. **流式输出实现**：
   - 使用 `useState` 管理 `renderedCount`（已渲染营养素个数）
   - `useEffect` 启动定时器，每 300ms 增加 `renderedCount`
   - 只有 `index < renderedCount` 的卡片才渲染，每个卡片添加 `animate-fadeIn` 动画

4. **参考标准区域**：灰色背景 `bg-gray-50`
5. **饮食建议区域**：浅蓝色背景 `bg-blue-50`
6. **高亮营养素**：边框橙色 `border-amber-400` + 浅色背景 `bg-amber-50`

### 路由流程
- 用户完成onboarding表单提交后 → 跳转 `/nutrition/{baby_id}`
- 用户确认营养分析 → 跳转 `/recipes/generate?baby_id={babyId}`

## 需求三：食谱展示改成 Tab 切换 + 流式输出

### 修改文件
- `frontend/app/recipes/[id]/page.tsx` - 完全重构展示逻辑

### 设计要点

1. **Tab 切换结构**：
   ```
   ┌─────────────────────────────────────────┐
   │ [周一] [周二] [周三] [周四] [周五] [周六] [周日] │
   ├─────────────────────────────────────────┤
   │                                         │
   │         当前选中那天的详细食谱              │
   │                                         │
   └─────────────────────────────────────────┘
   ```

2. **流式输出天**：
   - 使用 `useState` 管理 `renderedDays` 数组
   - 定时器每 400ms 添加一天到数组中
   - Tab 对未渲染完成的天置灰禁用
   - 用户可以点击已渲染的天自由切换

3. **样式**：
   - 选中态：粉色背景 `bg-pink-500 text-white`
   - 未选中态：灰色背景 `bg-gray-100`
   - 未渲染：半透明 `opacity-50` 禁用点击

## 需求四：「换一道菜」功能

### 后端新增/修改
- 新增 `backend/app/services/dish_replacer.py` - 替换菜品服务
- 修改 `backend/app/schemas.py` - 新增 `ReplaceDishRequest`
- 修改 `backend/app/routers/recipes.py` - 新增 `POST /api/recipes/{recipe_id}/replace-dish`

### 前端修改
- 修改 `frontend/app/recipes/[id]/page.tsx` - 添加按钮、点击处理、流式输出

### 设计要点

**后端设计**：
1. **替代菜品库**：按营养类型分类，每类有多道菜品
   - `iron_red_meat` - 红肉补铁类
   - `protein_poultry` - 禽肉高蛋白类
   - `seafood_fish` - 鱼虾水产类
   - `porridge` - 粥类主食
   - `vegetable_fruit` - 蔬果泥/蔬果块
   - `iron_rice` - 高铁米糊

2. **自动分类**：根据原菜品的食材和铁含量判断营养类型
3. **随机选择**：从同类中过滤掉原菜品同名，随机选一个
4. **原位更新**：直接更新原有 `RecipeDetail` 记录，不新增

**前端设计**：
1. **按钮位置**：菜品卡片右上角，粉色小圆按钮
2. **点击后**：按钮置灰显示加载动画，卡片半透明
3. **流式输出效果**：前端逐步构建菜品对象，每个字段之间停顿几百ms，模拟逐步生成
   - 先显示菜名 → 停顿
   - 逐个添加食材 → 每个食材停顿
   - 逐个添加烹饪步骤 → 每个步骤停顿
   - 添加营养信息 → 停顿
   - 添加图片
4. **完成后**：替换状态中的菜品数据

**API 设计**：
```
POST /api/recipes/{recipe_id}/replace-dish
Request Body:
{
  "day_of_week": 1,
  "meal_type": "lunch",
  "original_dish_id": 123
}

Response: Recipe (完整更新后的食谱)
```

**权限检查**：
- 验证食谱属于当前用户
- 验证原菜品属于该食谱
- 验证宝宝属于该用户
- 多层检查防止越权

## 样式动画

所有淡入动画统一在 `globals.css` 定义：
```css
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(10px); }
  to   { opacity: 1; transform: translateY(0); }
}
.animate-fadeIn {
  animation: fadeIn 0.5s ease-out forwards;
}
```
