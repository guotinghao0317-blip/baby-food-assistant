# 二期优化功能测试指南

## 测试环境
- 前端地址：`http://localhost:3000`
- 后端地址：`http://localhost:8000`

## 测试流程

### ✅ 测试一：营养分析页面（需求一+需求二）

**测试步骤**：
1. 登录系统
2. 点击开始，填写宝宝信息（选择6个月以上）
3. 提交后观察是否跳转到营养分析页面
4. 观察是否流式输出：
   - 宝宝信息卡片立刻显示
   - 每隔约300ms出现一张营养素卡片
   - 最后出现「确认并生成一周食谱」按钮

**预期结果**：
- [ ] 每个营养素卡片包含：名称+推荐值、参考标准（灰色背景）、饮食建议（浅蓝色背景）
- [ ] 铁和维生素D有高亮边框和「关键」标签
- [ ] 参考依据明确提到「WHO 婴幼儿营养指南 + 中国营养学会」
- [ ] 每个卡片淡入动画流畅
- [ ] 点击确认按钮能正确跳转到食谱生成

---

### ✅ 测试二：食谱页面 Tab 切换 + 流式输出（需求三）

**测试步骤**：
1. 完成营养分析，生成食谱后进入食谱详情页
2. 观察页面加载时是否一天一天依次出现Tab
3. 观察Tab是否可以点击切换

**预期结果**：
- [ ] Tab从左到右每隔400ms依次出现
- [ ] 点击不同Tab能正确切换显示对应天的食谱
- [ ] 页面不会过长（因为只显示当前选中一天）
- [ ] 移动端也能正常横向滚动Tab

---

### ✅ 测试三：「换一道菜」功能（需求四）

**测试步骤**：
1. 进入食谱详情页
2. 找到任意一道菜，点击右上角「🔄 换一道」
3. 观察加载状态
4. 观察生成效果
5. 查看新菜品是否替换成功

**预期结果**：
- [ ] 点击后按钮变成加载状态，显示「生成中」
- [ ] 新菜品内容逐步流式出现（菜名→食材逐个→步骤逐个→营养）
- [ ] 生成完成后，菜品更新为新菜
- [ ] 新菜和原菜**营养类型相同但食材不同**（比如原菜是牛肉土豆，替换后可能是猪肝西兰花，都是红肉补铁类）
- [ ] 餐次保持不变（原来午餐替换后还是午餐）

---

### ✅ 测试四：0-6个月特殊处理

**测试步骤**：
1. 重新填写宝宝信息，选择月龄 < 6个月
2. 查看营养分析页面

**预期结果**：
- [ ] 能量、蛋白质、脂肪、碳水都显示「主要从母乳/配方奶获取」
- [ ] 依然有铁、钙、维生素D的建议
- [ ] 正常生成食谱

---

## 异常情况测试

1. **未登录用户直接访问营养页** → 应该跳转登录
2. **访问不存在的baby_id** → 应该显示错误
3. **替换他人食谱** → 应该404错误（权限控制）
