## 1. 开发环境测试步骤（如何在本地测试）

### 1.1 启动本地开发环境

```bash
# 1. 进入前端项目目录
cd /Users/jiayindeng/宝宝辅食项目/frontend

# 2. 安装依赖（如未安装）
npm install

# 3. 启动开发服务器
npm run dev
```

### 1.2 在浏览器中打开本地地址
开发服务器启动后，会输出类似以下信息：
```
> Ready on http://localhost:3000
```
在 Chrome 浏览器中打开 `http://localhost:3000`

### 1.3 打开浏览器开发者工具

1. 按 `F12` 或右键点击页面选择「检查」打开开发者工具
2. 点击开发者工具左上角的**设备切换按钮（Toggle Device Toolbar）
3. 选择响应式模式，或选择不同的移动设备尺寸进行测试

### 1.4 测试不同屏幕尺寸

需要在以下常见移动设备尺寸测试：

- iPhone SE (375px × 667px)
- iPhone 12/13/14 (390px × 844px)
- iPhone 14 Pro Max (430px × 932px)
- iPad Mini (768px × 1024px)
- 横屏模式（旋转设备）

### 1.5 测试流程

按照 [章节3](#3-需要检查的页面和检查点) 逐个页面检查每个检查点。

## 2. 线上冒烟测试步骤（如何在生产环境验证）

### 2.1 准备工作

1. 使用移动设备（手机或平板）连接互联网
2. 准备线上网址（根据实际项目域名替换）：`https://your-domain.com

### 2.2 真实设备测试步骤

1. 打开手机浏览器（Safari 或 Chrome）
2. 访问线上地址，进入首页
3. 按照 [章节3](#3-需要检查的页面和检查点) 逐个页面进行测试
4. 测试完成后进行横竖屏切换，验证不同方向显示正常

### 2.3 Chrome DevTools 远程调试（可选）

如果需要在电脑上调试线上环境移动端问题：

1. 使用 USB 连接手机和电脑
2. 在电脑 Chrome 地址栏输入：`chrome://inspect`
3. 按照提示启用远程调试
4. 即可在电脑上调试手机浏览器中的页面

## 3. 需要检查的页面和检查点

### 3.1 页面列表

项目共有7个页面：

| 序号 | 页面路径 | 页面名称 |
|------|---------|---------|
| 1 | `/` | 首页 |
| 2 | `/login` | 登录页 |
| 3 | `/register` | 注册页 |
| 4 | `/onboarding` | 用户引导页 |
| 5 | `/nutrition/[baby_id]` | 营养分析页 |
| 6 | `/recipes/generate` | 食谱生成页 |
| 7 | `/recipes/[id]` | 食谱详情页 |

### 3.2 通用检查点（所有页面都需要检查）

| 检查项 | 检查点描述 |
|--------|-----------|
| ✅ 整体布局 | 页面内容不会超出视口，无横向滚动条（除非是可滚动组件如表格、卡片列表除外） |
| ✅ 字体大小 | 字体大小在移动端合适，文字不会过小，不清晰 |
| ✅ 点击区域 | 按钮、可点击元素的点击区域 >= 48px × 48px，符合移动端可用性标准 |
| ✅ 元素间距 | 元素之间间距合适，不会出现内容重叠 |
| ✅ 导航栏 | 顶部导航栏在移动端正常折叠/适配，没有被状态栏遮挡 |
| ✅ 输入框 | 输入框不会被软件盘遮挡，点击输入框页面能自动滚动调整可视区域 |
| ✅ 图片 | 图片自适应容器，不会溢出，保持宽高比正确 |
| ✅ 横竖屏切换 | 旋转屏幕后页面重新布局正确，内容显示正常 |
| ✅ 横向滚动 | 没有无效横向滚动条出现在页面整体 |
| ✅ 可滚动区域 | 需要滚动的内容区域滚动流畅，滚动正常，没有卡住现象 |
| ✅ 表单元素 | 表单元素尺寸适配，下拉框、单选框、复选框可正常点击 |

### 3.3 各页面特定检查点

#### 3.3.1 首页 (`/`)
- 检查项目：标题、按钮排列是否自适应
- 检查操作按钮在小屏幕是否正确堆叠

#### 3.3.2 登录页 (`/login`)
- 检查：输入框宽度适配屏幕宽度
- 检查：登录按钮可正常点击，区域合适

#### 3.3.3 注册页 (`/register`)
- 检查：表单输入项在移动端垂直排列
- 检查：各个表单项间距合适，能完整显示

#### 3.3.4 用户引导页 (`/onboarding`)
- 检查：步骤指示器正确适配屏幕尺寸
- 检查：下一步/完成按钮固定在底部，可见且可点击

#### 3.3.5 营养分析页 (`/nutrition/[baby_id]`)
- 检查：图表/数据图表在移动端可完整显示
- 检查：数据卡片在小屏幕上正确堆叠排列

#### 3.3.6 食谱生成页 (`/recipes/generate`)
- 检查：选项列表垂直排列正确
- 检查：生成按钮保持在可视区域
- 检查：生成结果内容不溢出，可正常滚动查看

#### 3.3.7 食谱详情页 (`/recipes/[id]`)
- 检查：食材列表、步骤列表排版正确
- 检查：步骤图片自适应宽度，图文排版清晰

## 4. 验收标准

### 4.1 必须满足条件

所有页面在 360px ~ 1024px 宽度范围内都满足以下要求：

1. **布局正确**：所有元素布局合理，元素不重叠，内容不超出屏幕
2. **交互可用**：所有按钮可正常点击，输入框可正常输入，没有点击区域符合要求
3. **内容完整**：所有文字、图片内容完整可见，没有被截断
4. **滚动正常**：只有需要滚动的区域可以滚动，页面本身不会出现横向滚动
5. **旋转正常**：横竖屏切换都能正确适配，不出现布局错乱

### 4.2 适配断点要求

设计适配断点规范：
- 小于 768px：移动端布局，内容单列显示
- 768px ~ 1024px：平板布局，适当利用空间
- 大于 1024px：保持桌面端原有布局不变

### 4.3 测试通过条件

- 所有 7 个页面全部通过测试，不存在影响使用的适配问题
- 不要求像素级完美，允许微小的间距或对齐问题不影响使用
- 严重问题：影响可用性或导致操作的问题必须修复后才能通过验收
