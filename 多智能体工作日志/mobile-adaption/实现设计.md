# 宝宝辅食项目 - 移动端适配技术设计

## 1. 需求概述

### 1.1 需求背景
当前项目已经完成核心功能开发，但现有页面在手机端浏览体验不好，需要专门做移动端适配。

### 1.2 适配范围
需要对以下7个页面进行手机端适配：
1. 首页 `/`
2. 登录页 `/login`
3. 注册页 `/register`
4. 宝宝信息收集页 `/onboarding`
5. 营养分析页 `/nutrition/[baby_id]`
6. 食谱生成页 `/recipes/generate`
7. 食谱详情页 `/recipes/[id]`

### 1.3 设计目标
- 适配手机屏幕宽度 360px - 414px
- 保持现有桌面端体验不受影响
- 不改变后端逻辑，不改变业务功能，仅调整前端样式

## 3. 适配方案设计

### 3.1 技术选型：Tailwind CSS 响应式断点

Tailwind CSS 默认断点正好满足需求，不需要自定义：

| 断点 | 前缀 | 起始宽度 | 适用场景 |
|------|------|----------|----------|
| sm | `sm:` | 640px | 手机横屏 |
| md | `md:` | 768px | 平板，**桌面端起点** |
| lg | `lg:` | 1024px | 大桌面 |

**设计原则**：
- 默认样式（无前缀）就是移动端样式 (< 768px)
- 使用 `md:` 前缀来覆盖桌面端样式
- 这样做是"移动优先"，符合 Tailwind 最佳实践

### 3.2 通用适配规则

#### 3.2.1 间距和内边距
| 元素 | 桌面端 | 移动端 | 类名示例 |
|------|--------|--------|----------|
| 容器左右 padding | `px-4 py-16` | `px-3 sm:px-4 py-8 md:py-16` | `container mx-auto px-3 sm:px-4 py-8 md:py-16` |
| 卡片内边距 | `p-8` | `p-4 md:p-6 md:p-8` | `bg-white rounded-xl shadow-lg p-4 md:p-8` |

#### 3.2.2 网格布局转换
当前代码很多地方使用了多列网格，需要：
- 移动端：单列 `grid-cols-1`
- 桌面端：原多列配置 `md:grid-cols-N`

示例：
```jsx
// 修改前
<div className="grid md:grid-cols-3 gap-8">

// 修改后（保持不变，已经正确 - 默认单列，md以上变3列）
<div className="grid grid-cols-1 md:grid-cols-3 gap-4 md:gap-8">
```

#### 3.2.3 文字大小调整
| 元素 | 桌面端 | 移动端 |
|------|--------|--------|
| 大标题 | `text-5xl` | `text-3xl md:text-5xl` |
| 中标题 | `text-3xl` | `text-2xl md:text-3xl` |
| 正文字号 | 保持不变 | 保持 `text-base`/`text-sm` 不小于14px |

#### 3.2.4 按钮可点击区域要求
WCAG 标准要求手机端可点击区域至少 **48x48px**

当前按钮已经满足大部分，但需要检查：
- 小按钮最小高度保证：`min-h-[48px]`
- 确保文字上下居中：`flex items-center justify-center`

### 3.3 特殊组件处理方案

#### 3.3.1 Tab栏横向滚动（食谱详情页）
当前问题：7天Tab在移动端会换行，影响体验

解决方案：
- 外层容器：`overflow-x-auto` + `flex` + `whitespace-nowrap`
- 允许用户横向滑动
- 添加轻量级滚动提示：`scrollbar-hide`（Tailwind 默认不隐藏，可以用 `overflow-x-auto pb-2`）

代码示例：
```jsx
<div className="flex gap-2 mb-6 border-b pb-2 overflow-x-auto">
  {days.map(day => (
    <button key={day} className="flex-shrink-0 ...">
      {DAYS[day - 1]}
    </button>
  ))}
</div>
```

#### 3.3.2 多选选项网格布局
当前问题：过敏源和食材多选在移动端两列太挤。

调整方案：
- 移动端：`grid-cols-1` 单列
- 平板：`sm:grid-cols-2` 两列
- 桌面：保持现有三列

```jsx
<div className="grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 gap-2">
```

#### 3.3.3 图片尺寸
当前食谱图片使用 `w-full h-48 object-cover`，这个在移动端已经没问题，可以保持。

#### 3.3.4 表单输入框
保持 `w-full px-4 py-2` 已经满足移动端需求，高度约 44px，加上上下间距已经够用。

### 3.4 禁止出现的问题
1. **不能出现横向滚动条**：所有内容必须限定在视口宽度内
2. **文字不能太小**：最小字号不小于 14px (`text-sm`)
3. **按钮不能太小**：可点击区域不小于 48x48px
4. **不能破坏桌面端**：所有修改都要用响应式前缀，桌面端保持原样

## 7. 风险与注意事项

### 7.1 技术风险
- **无**，因为只是样式调整，不涉及逻辑修改
- 如果出现问题，回滚也很简单

### 7.2 注意事项
1. 始终遵循"移动优先"，默认是移动端，`md:` 覆盖桌面端
2.不要删除原有桌面端样式，只是添加响应式前缀
3. 修改完一定要检查桌面端是否正常
4. 提交前一定要检查有没有横向滚动条

---

## 8. 总结

本次移动端适配是一个**纯前端样式调整需求**，技术方案成熟简单：
- 使用 Tailwind CSS 原生响应式断点，无需额外配置
* 遵循移动优先原则，通过 `sm:`, `md:` 前缀区分不同设备
- 对特殊的Tab组件使用横向滚动方案
- 总开发量不大，预计半天可以完成

所有修改都在现有文件上进行，不新增文件，不改变后端逻辑。
