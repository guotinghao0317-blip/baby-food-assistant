# 分步生成 - 后端实现文档

**日期**: 2026-04-20  
**项目**: 宝宝辅食助手

## 1. 概述

本实现为后端新增了**分步生成一周食谱**功能，支持两种模式：

1. **轮询模式**（前端当前使用）：
   - `POST /api/recipes/generate-start` - 初始化创建recipe
   - `GET /api/recipes/generate-next-day/{recipe_id}` - 每次调用生成一天，直接返回JSON
   - 前端轮询，每获取一天更新一次UI

2. **SSE流式模式**（保留后续使用）：
   - `POST /api/recipes/generate-step-by-step` - 保持连接，逐个推送SSE事件

主要特性：
- **双模式支持**: 同时支持轮询模式和SSE流式模式
- **菜品去重**: 每生成一天，都会检查菜品名称，避免重复
- **降级机制**: API 失败时自动使用缓存采样
- **完整事件流**: started -> day_started -> day_done -> finished (SSE模式)

## 3. API 接口定义

### 3.1 分步生成接口

**请求**:
```
POST /api/recipes/generate-step-by-step
Content-Type: application/json
Authorization: Bearer <token>

{
  "baby_id": 1
}
```

**响应**: Server-Sent Events 流

---

### 3.2 SSE 事件格式

| 事件类型 | 数据结构 | 说明 |
|----------|----------|------|
| `started` | `{"type": "started", "recipe_id": 123}` | 已创建空 Recipe 记录 |
| `day_started` | `{"type": "day_started", "day": 1}` | 开始生成第 N 天 |
| `day_done` | `{"type": "day_done", "day": 1, "details": [...]}` | 第 N 天生成完成，返回 4 道菜品 |
| `finished` | `{"type": "finished", "recipe_id": 123}` | 全部 7 天生成完成 |
| `error` | `{"type": "error", "message": "..."}` | 发生错误 |

## 4. 数据库说明

### 4.1 现有模型复用

**Recipe 表** (已存在 `status` 字段):
- `status = "generating"`: 生成中，刚创建空记录时
- `status = "completed"`: 全部生成完成

**RecipeDetail 表**: 无需修改，每天生成后批量插入 4 条记录

### 4.2 状态流转

```
1. 创建 Recipe: status = "generating"
   ↓
2. 循环 day 1-7: 逐条插入 RecipeDetail
   ↓
3. 全部完成: 更新 status = "completed"
```

## 5. 核心实现逻辑

### 5.1 去重机制

使用 `has_duplicate_dish()` 函数检查菜品名称相似度：
- 模糊匹配，包含相同关键词即视为重复
- 每生成一天，将 4 道菜名加入 `generated_dish_names` 列表
- 下一天生成时，传入该列表要求模型避开

### 5.2 降级处理

API 调用失败或格式错误时：
- 最多重试 2 次
- 仍失败则调用 `DishCacheService.sample_single_day()`
- 从已有菜品中随机采样，保证流程不中断

### 5.3 完整流程图

```
前端请求
  ↓
POST /generate-step-by-step
  ↓
验证宝宝信息 ──→ 失败则返回 404
  ↓
创建 Recipe (status=generating)
  ↓
发送 SSE: started {recipe_id}
  ↓
循环 day = 1 to 7:
  ├─ 发送 SSE: day_started {day}
  ├─ 调用火山引擎 API 生成
  │   └─ prompt 传入已生成菜名列表
  ├─ 检查去重
  ├─ API 失败则降级采样
  ├─ 保存 4 条 RecipeDetail 到数据库
  ├─ 菜名加入去重列表
  └─ 发送 SSE: day_done {day, details}
  ↓
更新 Recipe status = completed
  ↓
发送 SSE: finished {recipe_id}
  ↓
关闭连接
```

## 7. 使用示例

### 7.1 JavaScript 前端调用

```javascript
const eventSource = new EventSource('/api/recipes/generate-step-by-step', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + token
  },
  body: JSON.stringify({ baby_id: 1 })
});

eventSource.addEventListener('message', (e) => {
  const data = JSON.parse(e.data);
  
  switch(data.type) {
    case 'started':
      console.log('Recipe created:', data.recipe_id);
      break;
    case 'day_started':
      console.log('Generating day:', data.day);
      break;
    case 'day_done':
      console.log('Day', data.day, 'done:', data.details);
      break;
    case 'finished':
      console.log('All done!');
      eventSource.close();
      break;
    case 'error':
      console.error('Error:', data.message);
      eventSource.close();
      break;
  }
});
```

## 8. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 宝宝不存在 | 返回 HTTP 404 |
| 用户无权限 | 返回 HTTP 401 |
| API 调用失败 | 自动降级缓存采样，继续流程 |
| JSON 解析失败 | 重试 2 次，仍失败则降级 |
| 生成中途断连 | 已生成的数据保留在数据库，前端可重试 |
