## API测试

### 测试1：登录获取Token

```bash
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "18511285035@163.com", "password": "123456"}' \
  | jq '.access_token'
```

**保存token**: 将返回的token保存到环境变量

```bash
export TOKEN="your_token_here"
```

### 测试2：获取用户信息

```bash
curl http://localhost:8000/api/users/me \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.'
```

### 测试3：获取宝宝列表

```bash
curl http://localhost:8000/api/babies \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.'
```

### 测试4：生成食谱（流式）

```bash
curl -X POST http://localhost:8000/api/recipes/generate-stream \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"baby_id": 1}'
```

**观察输出**:
- 应该看到SSE格式的数据流
- 数据格式: `data: {"type": "...", ...}`

### 测试5：生成食谱（分步）

```bash
# 步骤1：开始生成
curl -X POST http://localhost:8000/api/recipes/generate-start \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"baby_id": 1}' \
  | jq '.recipe_id'

# 假设返回的recipe_id是13，保存它
export RECIPE_ID=13

# 步骤2：逐天生成
while true; do
  curl http://localhost:8000/api/recipes/recipe-status/$RECIPE_ID \
    -H "Authorization: Bearer $TOKEN" \
    | jq '.status'

  curl -X POST http://localhost:8000/api/recipes/generate-next-day/$RECIPE_ID \
    -H "Authorization: Bearer $TOKEN" \
    | jq '.'

  sleep 2
done
```

### 测试6：获取食谱详情

```bash
curl http://localhost:8000/api/recipes/$RECIPE_ID \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.'
```

### 测试7：更换菜品

首先获取要替换的菜品ID：

```bash
curl http://localhost:8000/api/recipes/$RECIPE_ID \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.details[0].id'
```

假设返回的detail_id是100，执行替换：

```bash
curl -X POST http://localhost:8000/api/recipes/$RECIPE_ID/replace-dish \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"detail_id": 100}'
```

## 数据库验证

### 验证1：查看最近的recipes

```bash
sqlite3 backend/babyfood.db "
  SELECT id, baby_id, status, created_at
  FROM recipes
  ORDER BY id DESC
  LIMIT 5
"
```

### 验证2：查看菜品数量

```bash
sqlite3 backend/babyfood.db "
  SELECT recipe_id, COUNT(*) as dish_count
  FROM recipe_details
  GROUP BY recipe_id
  ORDER BY recipe_id DESC
  LIMIT 5
"
```

### 验证3：验证meal_type去重（关键）

```bash
sqlite3 backend/babyfood.db "
  SELECT recipe_id, day_of_week, meal_type, COUNT(*) as count
  FROM recipe_details
  WHERE recipe_id = $RECIPE_ID
  GROUP BY recipe_id, day_of_week, meal_type
  HAVING count > 1
"
```

**期望结果**: 空集（无重复）

### 验证4：查看详细菜品

```bash
sqlite3 backend/babyfood.db "
  SELECT day_of_week, meal_type, dish_name
  FROM recipe_details
  WHERE recipe_id = $RECIPE_ID
  ORDER BY day_of_week, meal_type
"
```

### 验证5：统计覆盖天数

```bash
sqlite3 backend/babyfood.db "
  SELECT COUNT(DISTINCT day_of_week) as days
  FROM recipe_details
  WHERE recipe_id = $RECIPE_ID
"
```

**期望结果**: 7

## 常见问题排查

### 问题1：后端服务无法启动

```bash
# 检查端口占用
lsof -i :8000

# 如果被占用，杀掉进程
pkill -f "uvicorn main:app"

# 重新启动
cd backend && source venv/bin/activate
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 问题2：前端服务无法启动

```bash
# 清除node_modules缓存
rm -rf frontend/node_modules
cd frontend && npm install

# 重新启动
npm run dev
```

### 问题3：数据库锁定

```bash
# 有时SQLite会锁定，检查进程
ps aux | grep sqlite

# 如果需要，删除锁文件
rm -f backend/babyfood.db-shm backend/babyfood.db-wal
```

### 问题4：SSE连接断开

检查后端日志，看是否有错误：
- 检查token是否有效
- 检查baby_id是否存在
- 检查网络连接是否稳定
