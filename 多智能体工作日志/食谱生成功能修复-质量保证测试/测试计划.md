# 测试计划：食谱生成功能修复验证

## 测试概述

本次测试针对用户反馈的两个问题进行验证：
1. 食谱生成页显示为空，但控制台说数据加载成功
2. 数据只生成到周二，预期生成到周日

## 测试背景

### 已完成的修复

1. **后端meal_type去重逻辑**
   - 确保同一天、同一餐次只保存一道菜
   - 位置：`backend/routers/recipe_generation.py`

2. **前端实时菜品展示**
   - 在generate页面增加菜品实时展示区域
   - 实现打字机效果展示每道菜
   - 位置：`frontend/src/pages/recipe-generate.tsx`

3. **前端页面跳转优化**
   - 生成完成后自动跳转到详情页
   - 位置：`frontend/src/pages/recipe-generate.tsx`

## 测试范围

### 功能测试

#### 测试1：meal_type去重验证
- **测试目标**：验证每天每个餐次只保存一道菜
- **测试方法**：
  1. 调用生成食谱API
  2. 等待生成完成
  3. 查询数据库，验证每天每个餐次只有一道菜
  4. 如果发现重复，报告问题

#### 测试2：实时菜品展示功能
-   **测试目标**：验证生成过程中实时显示菜品
-   **测试方法**：
  1. 访问生成页面
  2. 调用生成API
  3. 观察页面是否实时显示每道菜
  4. 验证打字机效果是否正常

#### 测试3：页面自动跳转
- **测试目标**：验证生成完成后自动跳转
- **测试方法**：
  1. 调用生成API
  2. 等待finished事件
  3. 验证是否自动跳转到详情页
 4. 验证详情页能正常加载数据

## 测试重点

### 必须验证的场景

1. **数据完整性**：生成7天（周一到周日）的数据
2. **去重逻辑**：每天每餐只有一道菜
3. **实时展示**：生成过程中能看到菜品实时出现
4. **自动跳转**：生成完成后自动跳转到详情页
5. **用户体验**：无红色错误提示，操作流畅

### 可能的问题点

1. 去重逻辑未生效，出现重复菜品
2. 实时展示不工作，页面为空
3. 自动跳转失败，停留在生成页
4. 数据未完全生成（不到7天）
5. 网络错误提示
6. 控制台报错
