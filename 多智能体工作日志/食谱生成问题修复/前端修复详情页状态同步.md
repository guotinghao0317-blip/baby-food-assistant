# 前端修复详情页状态同步问题

## 问题描述

在食谱详情页 `/app/recipes/[id]/page.tsx` 中，周三到周日的标签页无法点击，即使数据库中已有这些天的数据。

## 问题分析

### 根本原因

1. **状态更新时机问题**：`fetchRecipe()` 函数中的 `setDays` 状态更新是异步的，可能在某些情况下状态没有正确同步到UI
2. **数据完整性检查不足**：没有充分检查数据格式和完整性，导致部分天的数据可能被错误地标记为未完成
3. **调试日志不够详细**：难以追踪数据加载和状态更新的过程，导致问题排查困难
4. **缓存与数据加载逻辑冲突**：`useEffect` 中的缓存加载和数据加载逻辑可能导致状态不一致

### 具体表现

- 数据库中已有7天的完整数据
- 但前端 `days` 状态中只有周一周二被标记为 `done`
- 周三到周日虽然状态为 `done`，但点击无响应

## 解决方案

### 1. 优化 `fetchRecipe()` 函数

**改进点：**
- 添加详细的数据完整性检查
- 验证 `day_of_week` 的类型和有效性
- 统计每天的菜品数量，检查数据是否完整
- 添加详细的调试日志

**核心代码：**
```typescript
// 按day_of_week分组，找出哪些天已有数据
const completedDays = new Set<number>()
const dayMealCounts: Record<number, number> = {}

recipeData.details.forEach((detail: any) => {
  if (detail.day_of_week && typeof detail.day_of_week === 'number') {
    completedDays.add(detail.day_of_week)
    dayMealCounts[detail.day_of_week] = (dayMealCounts[detail.day_of_week] || 0) + 1
  }
})

console.log('已生成的天数统计:', dayMealCounts)
console.log('已生成的天数列表:', Array.from(completedDays).sort((a, b) => a - b))

// 更新days数组状态：有数据的天设为done
setDays(prevDays => {
  const newDays = prevDays.map(d => {
    if (completedDays.has(d.day)) {
      console.log(`第 ${d.day} 天 (${DAYS[d.day - 1]}) 已完成，菜品数: ${dayMealCounts[d.day]}`)
      return { ...d, status: 'done' as const }
    }
    return d
  })
  return newDays
})
```

### 2. 优化 `useEffect` 初始化逻辑

**改进点：**
- 使用 `setTimeout` 延迟检查状态，确保 `fetchRecipe` 完成后再判断是否需要生成
- 检查实际完成的 `days` 状态，而不是依赖 `overallStatus`
- 添加详细的日志输出

**核心代码：**
```typescript
fetchRecipe().then(() => {
  setTimeout(() => {
    const currentDays = days
    const completedDays = currentDays.filter(d => d.status === 'done').length
    console.log(`当前已完成天数: ${completedDays}/7`)

    // 如果有缓存且所有天都已完成，不需要重新生成
    if (hasCache && completedDays === 7) {
      console.log('✅ 从缓存加载完成，不需要重新生成')
      return
    }

    // 如果还有未完成的天，继续生成
    if (completedDays < 7) {
      console.log(`⏳ 还有 ${7 - completedDays} 天需要生成，开始生成...`)
      startGeneration()
    }
  }, 100)
})
```

## 数据流程

### 数据加载流程

1. **页面初始化**
   - 尝试从 localStorage 加载缓存
   - 调用 `fetchRecipe()` 加载后端数据

2. **fetchRecipe() 执行**
   - 从后端 API 获取食谱数据
   - 验证数据格式和完整性
   - 更新 `recipe` 状态
   - 更新 `days` 状态，标记已生成的天为 `done`
   - 检查是否所有7天都已完成
   - 更新 `overallStatus` 状态
   - 如果已完成，保存缓存

3. **状态检查**
   - 延迟 100ms 检查 `days` 状态
   - 统计已完成的天数
   - 如果未完成，继续生成

4. **分步生成**
   - 循环检查后端状态
   - 生成下一天
   - 处理 SSE 事件，更新 UI
   - 全部完成后刷新数据

### 状态更新机制

- **recipe 状态**：存储从后端获取的完整食谱数据
- **days 状态**：存储每天的生成状态（pending/generating/done）
- **overallStatus 状态**：存储整体生成状态（idle/generating/finished/error）
- **selectedDay 状态**：存储当前选中的天数

**关键点：**
- `days` 状态的更新是异步的，需要在回调中使用最新状态
- 使用 `setTimeout` 确保状态更新完成后再进行后续操作
- 每次状态改变后都保存缓存
