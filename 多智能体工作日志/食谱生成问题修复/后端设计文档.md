# 食谱详情API优化 - 后端设计文档

## 问题描述
在食谱详情页中，周三到周日的标签页无法点击。经分析，后端`get_recipe()`端点返回的数据中`details`数组不完整，导致前端无法正确显示所有菜品数据。

## 根本原因
1. SQLAlchemy的lazy loading机制导致relationship关联数据未及时加载
2. 缺少显式的数据加载逻辑
3. 缺少数据验证和日志记录

## 解决方案

### 1. 修改文件
- `/Users/jiayindeng/baby-food-assistant/backend/app/routers/recipes.py`

### 2. 核心修改
在`get_recipe()`端点中添加：

#### 显式加载details数据
```python
# 显式加载所有菜品数据，确保返回完整的details数组
details = db.query(RecipeDetail).filter(
    RecipeDetail.recipe_id == recipe_id
).order_by(RecipeDetail.day_of_week, RecipeDetail.meal_type).all()

logger.info(f"查询到食谱 {recipe_id} 的菜品数量: {len(details)}")
```

#### 数据验证逻辑
```python
# 验证每个detail的必需字段
for detail in details:
    if not hasattr(detail, 'day_of_week') or detail.day_of_week is None:
        logger.warning(f"菜品 {detail.id} 缺少 day_of_week 字段")
    if not hasattr(detail, 'meal_type') or detail.meal_type is None:
        logger.warning(f"菜品 {detail.id} 缺少 meal_type 字段")
    if not hasattr(detail, 'dish_name') or detail.dish_name is None:
        logger.warning(f"菜品 {detail.id} 缺少 dish_name 字段")
```

#### 确保数据正确附加
```python
# 将details附加到recipe对象，确保序列化时包含
recipe.details = details
```

## API接口定义

### GET /api/recipes/{recipe_id}

**请求参数：**
- `recipe_id`: 路径参数，食谱ID
- 需要Bearer Token认证

**响应格式：**
```json
{
  "id": 1,
  "baby_id": 1,
  "week_start_date": "2026-04-20T00:00:00",
  "status": "completed",
  "created_at": "2026-04-20T10:00:00",
  "details": [
    {
      "id": 1,
      "day_of_week": 1,
      "meal_type": "breakfast",
      "dish_name": "南瓜米糊",
      "ingredients": [
        {"name": "南瓜", "amount": "50g"},
        {"name": "婴儿米粉", "amount": "20g"}
      ],
      "cooking_steps": [
        {"step": 1, "description": "将南瓜蒸熟"}
      ],
      "nutrition_info": {
        "calories": 100,
        "protein": 2.5,
        "iron": 0.5
      },
      "image_url": null
    },
    // ... 更多菜品（每天4餐，共7天28道菜）
  ]
}
```

**响应字段说明：**
- `id`: 食谱ID
- `baby_id`: 宝宝ID
- `week_start_date`: 周开始日期
- `status`: 食谱状态（generating/completed）
- `created_at`: 创建时间
- `details`: 菜品列表数组
  - `id`: 菜品ID
  - `day_of_week`: 星期几（1-7，周一-周日）
  - `meal_type`: 餐次类型（breakfast/lunch/dinner/snack）
  - `dish_name`: 菜品名称
  - `ingredients`: 食材列表
  - `cooking_steps`: 烹饪步骤
  - `nutrition_info`: 营养信息
  - `image_url`: 配图URL
