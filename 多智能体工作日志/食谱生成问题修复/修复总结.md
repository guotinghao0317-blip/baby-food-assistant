# SSE流式输出格式修复 - 总结

## 问题
用户访问 `/recipes/generate?baby_id=xxx` 后，食谱生成页显示为空。

## 根本原因
后端返回的菜品数据缺少 `id` 字段，导致前端无法正确渲染菜品列表。

## 解决方案
修改 `/backend/app/services/recipe_generator.py` 中的三个关键函数，在保存菜品到数据库后立即获取ID并包含在返回数据中。

## 修改的文件
- `/backend/app/services/recipe_generator.py` - 修改

## 修改的函数
1. `generate_next_day_sync` - 轮询模式使用的同步生成函数
2. `generate_next_day_stream` - 流式生成单天的函数
3. `generate_weekly_recipe_step_by_step` - 分步生成一周的主函数

## 核心改动
```python
# 修复前
detail = RecipeDetail(recipe_id=recipe_id, **detail_data)
db.add(detail)
saved_details.append(detail_data)

# 修复后
detail = RecipeDetail(recipe_id=recipe_id, **detail_data)
db.add(detail)
db.flush()  # 立即刷新以获取ID
db.refresh(detail)
saved_details.append({
    "id": detail.id,  # ⚠️ 关键：添加ID字段
    **detail_data
})
```

## 验证标准
✅ 所有事件都包含 `type` 字段
✅ `day_done` 事件包含 `day`、`details`、`generation_source`
✅ `details` 数组中每个菜品都包含 `id` 字段
✅ `finished` 事件包含 `recipe_id`、`total_dishes`
✅ `error` 事件包含 `message`
✅ 数据正确保存到数据库
✅ 前端能正确显示菜品
